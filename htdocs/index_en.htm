RelaxNGCC ( RELAX NG compiler compiler)

Last update May 3, 2002, by Daisuke OKAJIMA

RelaxNGCC is a tool for generating Java source code from a given RELAX NG grammar. By embedding code fragments in the grammar like yacc or JavaCC, you can take appropriate actions while parsing valid XML documents against the grammar.

Environment

To use RelaxNGCC, JRE 1.2 or later and an XML parser compliant with JAXP 1.2 are required. If you add Sun Multi-Schema XML Validator(MSV), it reports detailed error information about wrong input. MSV is not mandatory but I recommend you use RelaxNGCC with MSV. It is available from here.

Main features

Relaxer is another famous tool for generating Java source code from a given grammar, but RelaxNGCC is different from Relaxer at following points.

  • The code generated by RelaxNGCC can ignore unnecessary parts of the input XML document while Relaxer retrieves all information in the document. It is powerful especially when you want to extract necessary data from XML instances.
  • RelaxNGCC focuses on reading XML instances. Generation of XML documents from Java objects is outside the scope of RelaxNGCC.
  • RelaxNGCC makes programming easy for reading XML in comparison with writing raw DOM or SAX code.

About AttributesImpl.java

AttributesImpl.java in the relaxngcc.runtime package is used as an implementation of org.xml.sax.Attributes. It is based on the source code of org.xml.sax.helpers.AttributesImpl distributed with some products such as Xerces, but the original org.xml.sax.helpers.AttributesImpl has a bug in removeAttribute method. To avoid the bug, I modified the original and located at relaxngcc.runtime package. relaxngcc.runtime.AttributesImpl.java is in the public domain like the original for this reason. The bug has not been fixed in JRE1.4.
