RelaxNGCC ( RELAX NG compiler compiler)

Last update 2002/05/03, 岡嶋 大介May 3, 2002, by Daisuke OKAJIMA

　RelaxNGCCは、RELAX NGで書かれたスキーマからJavaのソースコードを生成するツールです。入力XMLの各エレメントやアトリビュートを読んだときのアクションを、yaccやJavaCCと同様の手法でユーザが直接スキーマに追加記述します。 RelaxNGCC is a tool for generating Java source code from a given RELAX NG grammar. By embedding code fragments in the grammar like yacc or JavaCC, you can take appropriate actions while parsing valid XML documents against the grammar.

実行環境Environment

　RelaxNGCCの実行にはJRE1.2以降が必要です。また、JAXP1.2経由でXMLパーサが利用可能でないといけません。さらに、Sun Multi-Schema XML Validator(MSV)があると、RELAX NG文法が誤っていたときに詳しい指摘をしてくれます。MSVは必須ではありませんが、一緒に入れておくことをお勧めします。MSVは、ここからダウンロードできます。 To use RelaxNGCC, JRE 1.2 or later and an XML parser compliant with JAXP 1.2 are required. If you add Sun Multi-Schema XML Validator(MSV), it reports detailed error information about wrong input. MSV is not mandatory but I recommend you use RelaxNGCC with MSV. It is available from here.

主な特徴Main features

　スキーマからソースコードを生成するツールとしてはRelaxerが有名ですが、RelaxNGCCは次のような違いがあります。 Relaxer is another famous tool for generating Java source code from a given grammar, but RelaxNGCC is different from Relaxer at following points.

  • RelaxNGCCは、もとのXML文書の木構造にとらわれない処理が簡単です。RelaxerがXML文書全体について各部分に対応するオブジェクトを生成するのに対し、XML文書から必要なデータだけを抽出・加工する処理が得意です。The code generated by RelaxNGCC can ignore unnecessary parts of the input XML document while Relaxer retrieves all information in the document. It is powerful especially when you want to extract necessary data from XML instances.
  • RelaxNGCCは、XML文書を読み込むだけの機能です。XML文書を出力する機能はありません。XML->Javaの一方通行のツールです。RelaxNGCC focuses on reading XML instances. Generation of XML documents from Java objects is outside the scope of RelaxNGCC.
  • 一からDOMやSAXのプログラムを書くよりも簡単にXMLを読む機能が実装できます。RelaxNGCC makes programming easy for reading XML in comparison with writing raw DOM or SAX code.

AttributesImpl.javaについて注意About AttributesImpl.java

　relaxngcc.runtimeパッケージにあるAttributesImpl.javaはorg.xml.sax.Attributesの実装として使っています。このソースコードはXerces等とともに配布されているorg.xml.sax.helpers.AttributesImplのソースコードを元にしていますが、現在のところこれにはremoveAttributeメソッドにバグがあるため、それを避ける目的でバグを修正したものをrelaxngcc.runtimeパッケージに配置して使用しています。なので当面の間、relaxngcc.runtime.AttributesImpl.javaだけはorg.xml.sax.helpers.AttributesImplに倣ってpublic domainのライセンスとします。なおこのバグはJRE1.4でも同様の模様です。 AttributesImpl.java in the relaxngcc.runtime package is used as an implementation of org.xml.sax.Attributes. It is based on the source code of org.xml.sax.helpers.AttributesImpl distributed with some products such as Xerces, but the original org.xml.sax.helpers.AttributesImpl has a bug in removeAttribute method. To avoid the bug, I modified the original and located at relaxngcc.runtime package. relaxngcc.runtime.AttributesImpl.java is in the public domain like the original for this reason. The bug has not been fixed in JRE1.4.
